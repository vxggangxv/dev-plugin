# 코드 리뷰 체크리스트

## 리뷰 접근법

코드를 읽을 때 **"이 코드가 어떻게 깨질 수 있는가"**를 먼저 생각한다.

1. **멘탈 실행**: 정상 입력, 엣지 케이스, null 값으로 코드를 머릿속에서 실행
2. **암묵적 가정 찾기**: 코드가 전제하는 가정(데이터 크기, 순서, 타입, 동시성)을 식별
3. **실패 시나리오 우선**: 네트워크 실패, 타임아웃, 부분 실패, 동시 요청 상황을 상상

---

## [blocker] 반드시 수정 — 머지 차단

| 카테고리 | 점검 항목 |
|---------|----------|
| **정확성: 논리 오류** | 조건문의 논리 연산자(`and`/`or`) 오용, 부정 조건 실수 (`not A and B` vs `not (A and B)`), if-else 분기 누락 |
| **정확성: Null 참조** | DB 쿼리/API 응답이 None/null일 수 있는데 검증 없이 접근, Optional chaining 없는 중첩 객체 접근 |
| **정확성: 비동기** | `await` 누락, 트랜잭션 중간 실패 시 롤백/정리 미보장, check-then-act 패턴의 race condition |
| **정확성: 경계 조건** | 루프 종료 조건 오류 (`<` vs `<=`), 배열 인덱스 범위 초과, 빈 컬렉션 미처리 |
| **정확성: 상태 전이** | 잘못된 상태로 진입할 수 있는 경로 존재, 공유 상태 변경이 다른 코드에 영향 |
| **정확성: 수치 연산** | 부동소수점 `==` 비교, 정수 나눗셈 truncation, 금액 계산에 float 사용 |
| **보안: 인젝션** | SQL 문자열 결합 쿼리, XSS 미이스케이프 출력, OS 커맨드 인젝션 |
| **보안: 시크릿** | 하드코딩된 API 키, 비밀번호, 토큰, 시크릿 |
| **보안: 인증/인가** | 상태 변경 엔드포인트에 권한 체크 누락, 서버 사이드 인가 없음 |
| **보안: 입력** | 미검증 사용자 입력이 리다이렉트/파일경로/eval에 직접 사용 |
| **데이터** | 데이터 유실/손상 위험 (롤백 불가 마이그레이션, 무조건 DELETE, 트랜잭션 누락) |
| **호환성** | public API 브레이킹 체인지 (마이그레이션 경로 없음) |

## [issue] 수정 권장 — 머지 전 해결

| 카테고리 | 점검 항목 |
|---------|----------|
| **정확성: 반환값** | 함수 반환값이 None/null일 수 있는데 호출자가 미처리, 딕셔너리 존재하지 않는 키 접근 |
| **정확성: 타입** | 문자열↔숫자 암묵적 변환 의존, 시간/날짜 연산에서 타임존/DST 미고려, 정수 오버플로우 가능성 |
| **정확성: 비동기** | Promise/async 에러 핸들링 누락, 병렬 요청 결과 순서 의존, 타임아웃 미설정 |
| **정확성: 상태** | mutable 객체를 인자로 전달 후 의도치 않게 변경, 캐시 무효화 조건 오류, 재시도 시 멱등성 미보장 |
| **정확성: 요구사항** | 요구사항/티켓의 조건이 코드에 누락, 의도하지 않은 기능 추가 |
| **에러 처리** | I/O 작업에 에러 핸들링 누락, 빈 catch 블록, 부분 실패 시 일관성 미보장 |
| **에러: 리소스** | 에러 발생 시 리소스(파일 핸들, DB 커넥션, 락) 미해제 (finally/context manager 누락) |
| **테스트** | 새 기능/버그 수정에 테스트 없음, 코드 변경 시 실제로 실패하지 않는 무의미한 테스트 |
| **성능: DB** | N+1 쿼리, LIMIT 없는 SELECT, 커넥션 누수, 인덱스 없는 쿼리 패턴 |
| **성능: 루프** | 루프 내 DB/API 호출, 루프 내 문자열 결합 |
| **동시성** | 공유 가변 상태에 동기화 없음, 데드락 가능한 락 획득 순서 |
| **로깅** | PII(이메일, 비밀번호, 카드번호 등)가 로그에 노출 |
| **입력 검증** | 외부 입력(API 파라미터, 폼 입력)에 유효성 검증 누락 |
| **미완성** | 불완전한 변경 — 기능은 있지만 테스트/마이그레이션/문서가 빠짐 |

## [suggestion] 개선 권장 — 선택적

| 카테고리 | 점검 항목 |
|---------|----------|
| **복잡도** | 50줄 이상 함수, 3단계+ 중첩, God 클래스 (500줄+) |
| **네이밍** | 의도가 불명확한 변수/함수명 |
| **중복** | 복붙된 유사 코드 블록 (공통 함수 추출 가능) |
| **관찰성** | 실패 시 디버깅을 위한 로그/메트릭/트레이스 부재 |
| **테스트 커버리지** | 엣지 케이스(빈 입력, 경계값, 에러 경로) 미커버, 비결정적(flaky) 테스트 요소 |
| **API 설계** | 일관성 없는 네이밍, 비멱등 쓰기 작업, 불명확한 에러 메시지 |

## [nit] — 리뷰에서 제외 (린터/포매터에 맡김)

- 포맷팅, 공백, import 순서, 세미콜론
- 개인 스타일 선호 (정답 없는 취향 차이)
- 현재 변경과 무관한 기존 기술 부채
- 변수명 케이스 스타일 (camelCase vs snake_case)

## 리뷰 원칙

- **WHY를 설명**: "이게 문제다"가 아니라 "이렇게 하면 X가 발생할 수 있다"
- **구체적 수정안 제시**: issue 이상은 반드시 수정 코드/방법 포함
- **15개 이내**: 코멘트가 너무 많으면 리뷰 피로. 심각도 높은 것 위주
- **칭찬도 포함**: 잘 작성된 부분이 있으면 `[praise]`로 언급
- **기존 문제 지적 금지**: 현재 변경과 무관한 기존 코드 문제는 별도 이슈로
