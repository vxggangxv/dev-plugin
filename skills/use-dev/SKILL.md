# use dev - 개발 에이전트 실행

작업 영역을 자동 판별하여 적절한 개발 에이전트를 선택하고 실행한다.

## 실행 절차

### 1. 작업 컨텍스트 파악

현재 세션에서 다음을 확인:
- specs 폴더의 spec.md가 있는지
- 세션에서 논의된 작업 내용
- 지정된 작업 대상 파일

### 2. 작업 영역 자동 판별

| 작업 영역 | 자동 선택 에이전트 | 판별 기준 |
|----------|------------------|----------|
| 백엔드 | `backend-dev` | API, DB, 서버 로직, 인증, 데이터 처리 |
| 웹 프론트 | `frontend-web-dev` | React/Vue/Next.js 컴포넌트, 페이지, 스타일링 |
| 앱 프론트 | `frontend-app-dev` | React Native/Flutter 스크린, 앱 컴포넌트 |
| 풀스택 (독립) | 병렬 실행 | 백엔드 + 프론트 동시 개발 가능 |
| 풀스택 (의존) | 순차 실행 | 백엔드 결과 → 프론트 입력 필요 |

### 3. 리서치 필요 여부 판단

기존 코드 패턴 파악이 필요하면 개발 전 탐색 에이전트를 선행:
```
[codebase-explorer로 구조 파악]
     ↓
[docs-researcher로 문서 확인 (필요시)]
     ↓
[개발 에이전트 실행]
```

### 4. 에이전트 실행

`~/.claude/skills/use-agents/references/AGENT_SYSTEM.md`의 실행 방법을 따른다.

**단일 영역 작업**:
```
[영역 판별: API 작업] → [backend-dev 실행]
```

**복합 영역 작업 (독립적)**:
```
[동시 실행: backend-dev + frontend-web-dev]
     ↓
[두 결과 수집 후 통합 완료 보고]
```

**복합 영역 작업 (의존성)**:
```
[1. backend-dev로 API 개발] → 완료 대기
     ↓
[2. frontend-web-dev로 UI 개발]
     ↓
[통합 완료 보고]
```

### 5. 완료 보고

```markdown
---
## use dev 완료

**기능**: [기능명]
**작업 영역**: [백엔드/웹 프론트/앱 프론트/풀스택]
**사용된 에이전트**: [에이전트명]

### 수정된 파일
- [파일 목록]

### 검증
- [테스트/타입체크/린트 결과]
---
```

## 주의사항

- 기존 프로젝트의 코드 패턴/컨벤션 준수
- **병렬 실행 우선**: 독립적 작업은 동시 실행으로 속도 향상
- **의존성 체크**: 작업 간 의존성 있을 때만 순차 실행
- 각 에이전트 완료 후 다음 에이전트에게 컨텍스트 전달 (순차 실행 시)
