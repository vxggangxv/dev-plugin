# Clean Code Review and Improvement

작업된 코드를 프로젝트 가이드라인에 맞춰 종합적으로 리뷰하고 개선합니다.

## 리뷰 프로세스

다음 순서로 체계적인 리뷰를 수행하세요:

### 1. 변경된 파일 확인

먼저 `git status`와 `git diff`를 실행하여 작업된 파일들을 확인합니다.

### 2. CLAUDE.md 가이드라인 준수 확인

- 각 파일이 올바른 위치에 생성되었는지 확인
- 각 파일에 대해 검증

### 4. 타입스크립트 에러 체크

다음 명령어로 타입 에러를 확인:

```bash
npx tsc --noEmit
```

발견된 모든 타입 에러를 수정합니다.

### 5. ESLint 검사

```bash
npm run lint
```

린트 확인하고 error만 수정합니다. warning은 수정하지 않습니다.

### 6. 웹개발에서는 브라우저 호환성 체크 (Safari, Chrome)

작업된 내용에 대해 Safari, Chrome 브라우저 호환성을 체크합니다.

### 7. 로직 및 시나리오 크로스체크

각 파일의 비즈니스 로직을 검토:

- 엣지 케이스 처리
- 에러 핸들링
- 상태 관리 로직의 일관성
- API 호출 및 응답 처리
- 컴포넌트 간 데이터 흐름

### 8. 민감 파일 및 환경 값 점검

**민감 파일 포함 여부**
- `.env`, `.env.*` 파일이 변경사항에 포함되었는지
- `credentials.*`, `secrets.*`, `*.pem`, `*.key` 포함 여부
- 포함되어 있으면 경고

**로컬 환경 값 유출**
- `0.0.0.0`, `127.0.0.1`, `localhost` 등 로컬 도메인/IP가 코드에 하드코딩된 경우
- 설정 파일(`config`, `settings`, `.env.example`)이나 테스트 코드에서의 사용은 허용

**공통 코드 의도치 않은 변경**
- `APIRouter()` 파라미터 변경 (prefix, tags 등)
  - 예: `APIRouter(prefix="/v1/subscriptions")` → `APIRouter(prefix="/v1/subs")` 같은 변경
- 공통 미들웨어, 의존성 주입 함수 변경
- `__init__.py`의 export 변경

**설정/인프라 파일 의도치 않은 변경**
- `alembic.ini`, `pyproject.toml`, `requirements.txt`, `Dockerfile` 등
- 현재 작업과 관련 없는 설정 변경이면 경고

### 9. 불필요한 코드 제거

다음을 확인하고 제거:

- 사용하지 않는 import
- 사용하지 않는 변수/함수
- 주석 처리된 코드 블록 (`# TODO`, `# FIXME`는 허용)
- 디버깅 코드: `console.log()`, `print()`, `breakpoint()`, `pdb.set_trace()` 등
- 중복된 코드

### 10. 개선 사항 적용

발견된 모든 문제를 수정하고:

- 각 파일별로 개선 사항을 명확히 설명
- 변경 사항을 단계적으로 적용
- 수정 후 다시 타입 체크 및 린트 실행
