# 디버깅 스킬

debugger 에이전트를 spawn하여 에러를 체계적으로 분석하고 해결한다.

---

## 실행 절차

1. 에러 정보 수집: 사용자가 제공한 에러 메시지, 스택 트레이스, 재현 조건을 정리한다.
2. `debugger` 에이전트 파일을 읽는다.
3. Agent 도구를 사용하여 debugger 에이전트를 spawn한다.

```
Agent 도구 파라미터:
- subagent_type: "general-purpose"
- description: "debugger - 에러 분석 및 해결"
- prompt: |
    [debugger 에이전트 내용]

    ---
    ## 디버깅 대상

    **에러 메시지**:
    [사용자가 제공한 에러 메시지]

    **스택 트레이스**:
    [있으면 포함]

    **재현 조건**:
    [있으면 포함]

    **작업 디렉토리**:
    [현재 작업 디렉토리 경로]

    **추가 컨텍스트**:
    [사용자가 제공한 추가 정보]
```

4. 에이전트의 디버깅 결과를 사용자에게 전달한다.

## 결과 후 행동

- 해결책이 제시되면 사용자에게 수정 적용 여부를 확인한다.
- 수정 적용 시 해당 개발 에이전트(backend-dev, frontend-web-dev 등)를 실행할 수 있다.
